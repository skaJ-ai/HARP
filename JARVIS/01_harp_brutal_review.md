# HARP 냉정 리뷰

## 이 툴이 주는 가치
HARP의 핵심 가치는 단순한 "ChatGPT wrapper"가 아니라,
**HR 담당자가 AI와 대화하며 회사 표준 형식의 보고서 초안을 만들고, 그 과정이 재사용 가능한 자산으로 쌓이는 private-first 작업공간**이라는 점입니다.

즉, 팔고 있는 것은 LLM 호출이 아니라 다음의 운영체계입니다.
- 템플릿 기반 시작
- 체크리스트 기반 인터뷰
- 실시간 캔버스 반영
- 결과물 표준화
- 과거 작업 재활용
- draft → final → promoted asset 흐름

이건 실제 HR 실무에서 의미가 있습니다. HR은 자유형 글쓰기가 아니라 **형식·톤·근거·재사용성**이 중요한 영역이기 때문입니다.

## 무엇이 좋은가
### 1. 사용자 타깃이 선명함
대상 사용자가 HR 담당자로 명확하고, 대표 use case도 선명합니다.
- 주간 HR 현황 보고
- 교육 운영 결과 요약
- 제도 검토 초안

### 2. "AI가 먼저 묻는다" UX가 실용적임
대부분의 현업은 프롬프트를 잘 못 씁니다. HARP는 이걸 인정하고,
- 체크리스트 기반 인터뷰
- 빠진 항목 추적
- 방법론 카드 제안
- 준비도 퍼센트
로 사용 진입장벽을 낮춥니다.

### 3. 컨텍스트 설계 철학이 맞음
핵심은 모델보다 컨텍스트라는 판단이 옳습니다.
- 최근 deliverable 요약
- source context
- example text 스타일 레퍼런스
- 하이브리드 retrieval
이런 요소는 실무형 AI 제품에서 맞는 방향입니다.

### 4. 코드/문서 정합성이 좋음
README/SDD와 실제 구현이 꽤 일치합니다. 이건 보기보다 큰 장점입니다.

## 무엇이 아쉬운가
### 1. 차별화가 아직 약함
냉정하게 말하면 현재 HARP는
**도메인 특화된 AI 문서 생성기 + 워크스페이스**입니다.
좋은 구현이지만, 아직은 대형 워크스페이스/문서 AI 제품 대비 압도적 방어력이 보이지 않습니다.

### 2. 구현된 템플릿 정체성이 다소 흔들림
문서상 3개 대표 시나리오가 강하지만, 코드상 템플릿은 `analysis / planning / result / status` 로 더 범용적입니다.
즉, 제품 메시지는 HR vertical인데, 구현은 약간 generic document copilot 쪽으로 이동해 있습니다.

### 3. 품질평가가 약함
confidence/cited 메타는 useful하지만, 아직은 모델 자기신고에 가깝습니다.
실제 신뢰성을 만들려면 다음이 필요합니다.
- source-to-section 근거 추적
- unsupported claim 경고
- human review feedback loop
- template별 품질 지표

### 4. 개인 생산성 툴을 넘는 조직 운영 기능이 없음
협업, 승인, 코멘트, 버전 비교, 리뷰 요청, 자산 승인 같은 조직 기능이 거의 비어 있습니다.

## 최종 판정
HARP는 **방향 좋은 vertical AI product**입니다.
다만 아직은
- 좋은 팀 내부 파일럿
- 잘 만든 실험적 워크벤치
에 가깝고,
진짜 상용 제품이 되려면 **문서 초안 생성**보다 **문서 운영체계** 쪽을 더 강하게 만들어야 합니다.
