# HARP 경쟁 분석: DoXA 대비 차별점

> 작성일: 2026-03-27
> 목적: HARP의 포지셔닝을 사내 유사 서비스 대비 명확히 하기 위한 분석 문서

---

## 1. 비교 대상: DoXA

DoXA는 사내 문서 인텔리전스 서비스로, 3개 핵심 기능을 제공한다.

| 기능 | 설명 |
|------|------|
| **Parser** | 사내에서 사용되는 다양한 타입의 문서를 AI 모델에 친화적인 데이터로 변환 |
| **RAG** | 문서 내 텍스트와 이미지를 함께 이해하여 사용자의 질문에 연관된 문서를 검색 |
| **KIE** | 문서 양식별 지정된 정보를 추출하여 구조화된 JSON 형태로 제공 |

---

## 2. 핵심 차이: 작업 방향

| | DoXA | HARP |
|---|---|---|
| **방향** | 기존 문서 → 구조화된 데이터 (입력 처리) | 흩어진 지식 → 새 표준 문서 (출력 생성) |
| **질문** | "이 문서에서 무엇을 꺼낼까?" | "이 사람이 무엇을 써야 하는데 빠진 게 뭘까?" |
| **최종 산출물** | JSON, 검색 결과, 파싱된 데이터 | 회사 표준 형식의 보고서 초안 |
| **사용자** | 시스템/개발자 (API 소비자) | HR 담당자 (비개발자 엔드유저) |

**DoXA는 "문서를 읽는 AI", HARP는 "문서를 쓰는 AI"다.**

---

## 3. 기능별 대조

| DoXA 기능 | 하는 일 | HARP와의 관계 |
|-----------|---------|---------------|
| **Parser** | 문서 → AI 친화 데이터 변환 | HARP는 문서를 읽지 않는다. 담당자의 대화와 붙여넣기 자료로부터 문서를 **만든다** |
| **RAG** | 질문 → 관련 문서 검색 | HARP의 검색은 workspace 내 **자기가 만든** 산출물과 근거자료만 대상. 전사 문서 검색이 아님 |
| **KIE** | 양식별 정보 추출 → JSON | HARP는 추출이 아니라 **구조화된 섹션을 생성**. section-meta(confidence/cited)로 품질을 태깅 |

---

## 4. 포지셔닝

- **DoXA** = 문서 인프라/미들웨어. 이미 존재하는 문서를 기계가 이해할 수 있게 변환하는 파이프라인.
- **HARP** = 엔드유저 작업 도구. 아직 존재하지 않는 보고서를 HR 담당자가 AI와 대화하며 만드는 워크스페이스.

---

## 5. 보완 관계 (경쟁이 아닌 연동 가능성)

DoXA가 처리한 결과물이 HARP의 **근거자료(source)** 로 들어올 수 있다.

```
[DoXA Parser] 교육 설문 PDF → 구조화 텍스트
  ↓
[HARP source] 세션에 근거자료로 첨부
  ↓
[HARP 산파술] AI가 체크리스트 기반으로 인터뷰
  ↓
[HARP deliverable] 교육 운영 결과 보고서 초안
```

### 연동 시나리오 예시

| 시나리오 | DoXA 역할 | HARP 역할 |
|----------|-----------|-----------|
| 교육 결과 보고 | 설문 PDF를 파싱하여 수치 데이터 추출 | 추출된 데이터를 근거자료로 받아 보고서 초안 생성 |
| 제도 검토 | 기존 제도 문서에서 관련 조항 RAG 검색 | 검색된 조항을 참고하여 검토 의견서 작성 |
| 주간 현황 | KIE로 각 팀 주간 보고에서 핵심 지표 추출 | 추출된 지표를 종합하여 HR 전체 주간 보고 생성 |

---

## 6. 요약

겹치는 영역이 없다. DoXA는 문서 처리 인프라이고, HARP는 문서 생성 워크스페이스다. 장기적으로는 DoXA의 Parser/RAG 출력을 HARP의 source로 자동 공급하는 파이프라인이 가장 높은 시너지를 만든다.
